<AgentArchitecture />

Traditional automation follows rigid rules: "if X, then Y." AI agents break this limitation. They observe context, reason about what to do, take action using tools, and learn from the result. This makes them perfect for automations that require judgment, natural language understanding, or handling edge cases.

If you've taken the [Agentic Web](/agentic-web) course, you understand the theory. This module is about putting agents to work as practical automators.

<Accordion type="single" collapsible className="w-full mt-4">
    <AccordionItem value="item-1">
        <AccordionTrigger className="text-lg text-left">The Agent Loop: Observe → Think → Act → Learn</AccordionTrigger>
        <AccordionContent className="prose dark:prose-invert max-w-none text-base">
            <p>Every AI agent runs a loop:</p>
            <ol>
                <li><strong>Observe:</strong> Read the current context. user request, data from APIs, error messages, previous results.</li>
                <li><strong>Think:</strong> Plan the next action. The LLM reasons about what tools to call, what order to do things in, and what could go wrong.</li>
                <li><strong>Act:</strong> Execute the plan. call an API, run a database query, send a message, write a file.</li>
                <li><strong>Learn:</strong> Evaluate the result. Did it work? If not, adjust and loop again with new context.</li>
            </ol>
            <p><strong>Why this matters for automation:</strong> Traditional automation breaks when it encounters something unexpected. An AI agent can handle exceptions, ambiguous data, and novel situations by reasoning about them instead of following a fixed script.</p>
            <p><strong>Example:</strong> A rule-based automation fails when a job title is in a new format. An AI agent reads the title, understands it's still a "Senior Engineer" role despite unusual formatting, and processes it correctly.</p>
        </AccordionContent>
    </AccordionItem>

    <AccordionItem value="item-3">
        <AccordionTrigger className="text-lg text-left">Claude & GPT: Tool-Calling Agents</AccordionTrigger>
        <AccordionContent className="prose dark:prose-invert max-w-none text-base">
            <p>Claude and GPT support <strong>tool calling</strong>. you define functions the model can invoke, and the model decides when to use them based on the conversation.</p>
            <p><strong>Building an automation agent with tool calling:</strong></p>
            <pre><code>{`// Define tools the agent can use
const tools = [
  {
    name: "search_jobs",
    description: "Search the job database by keyword and location",
    parameters: {
      keyword: { type: "string" },
      location: { type: "string" },
    },
  },
  {
    name: "post_to_telegram",
    description: "Send a formatted message to the Telegram channel",
    parameters: {
      message: { type: "string" },
    },
  },
  {
    name: "update_spreadsheet",
    description: "Add a row to the tracking spreadsheet",
    parameters: {
      data: { type: "object" },
    },
  },
];

// The agent decides which tools to call
const response = await anthropic.messages.create({
  model: "claude-sonnet-4-20250514",
  tools,
  messages: [{
    role: "user",
    content: "Find all new Solidity jobs posted today, " +
      "post a summary to Telegram, and log them in the spreadsheet."
  }],
});`}</code></pre>
            <p>The model will call <code>search_jobs</code>, process the results, call <code>post_to_telegram</code> with a formatted message, then call <code>update_spreadsheet</code>. all autonomously.</p>
            <p>For deeper coverage of how prompting drives agent behavior, see the [Prompt Engineering](/prompt-engineering-101) course.</p>
        </AccordionContent>
