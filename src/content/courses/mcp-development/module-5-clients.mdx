<ClientEcosystem />

An MCP server is useless without a client. The good news is that MCP adoption has exploded: all major AI coding tools and assistants now support it. This module covers how to connect your server to each of them.

<Accordion type="single" collapsible className="w-full mt-4">
    <AccordionItem value="item-1">
        <AccordionTrigger className="text-lg text-left">Claude Desktop Configuration</AccordionTrigger>
        <AccordionContent className="prose dark:prose-invert max-w-none text-base">
            <p>Claude Desktop is the reference MCP client. It supports all MCP features and is the best environment for testing.</p>
            <p><strong>Configuration file location:</strong></p>
            <ul>
                <li><strong>macOS:</strong> <code>~/Library/Application Support/Claude/claude_desktop_config.json</code></li>
                <li><strong>Windows:</strong> <code>%APPDATA%\Claude\claude_desktop_config.json</code></li>
            </ul>
            <p><strong>Configuration format:</strong></p>
            <pre><code>{`{
  "mcpServers": {
    "my-server": {
      "command": "node",
      "args": ["dist/index.js"],
      "cwd": "/path/to/my-server",
      "env": {
        "DATABASE_URL": "postgresql://...",
        "API_KEY": "sk-..."
      }
    },
    "another-server": {
      "command": "npx",
      "args": ["-y", "@some-org/mcp-server-name"]
    }
  }
}`}</code></pre>
            <p><strong>Key fields:</strong></p>
            <ul>
                <li><code>command</code>. The executable to run (node, npx, python, etc.)</li>
                <li><code>args</code>. Command-line arguments</li>
                <li><code>cwd</code>. Working directory for the server process</li>
                <li><code>env</code>. Environment variables (use this for API keys and secrets)</li>
            </ul>
        </AccordionContent>
    </AccordionItem>
    <AccordionItem value="item-2">
        <AccordionTrigger className="text-lg text-left">Cursor & Windsurf Integration</AccordionTrigger>
        <AccordionContent className="prose dark:prose-invert max-w-none text-base">
            <p>Cursor supports MCP servers natively through its settings. The configuration is similar to Claude Desktop but managed through the UI.</p>
            <p><strong>Setting up in Cursor:</strong></p>
            <ol>
                <li>Open Cursor Settings → Features → MCP</li>
                <li>Click "Add MCP Server"</li>
                <li>Provide the server name, command, and arguments</li>
                <li>Set any required environment variables</li>
            </ol>
            <p>Cursor also supports a <code>.cursor/mcp.json</code> file in your project root for per-project MCP configuration:</p>
            <pre><code>{`{
  "mcpServers": {
    "project-db": {
      "command": "npx",
      "args": ["tsx", "mcp/db-server.ts"],
      "env": {
        "DATABASE_URL": "postgresql://localhost:5432/myapp"
      }
    }
  }
}`}</code></pre>
            <p>This means you can ship MCP configurations with your codebase. team members automatically get the same tools when they open the project.</p>
        </AccordionContent>
    </AccordionItem>
    <AccordionItem value="item-3">
        <AccordionTrigger className="text-lg text-left">VS Code with GitHub Copilot</AccordionTrigger>
        <AccordionContent className="prose dark:prose-invert max-w-none text-base">
            <p>VS Code supports MCP through its GitHub Copilot Chat integration. Configuration uses the <code>.vscode/mcp.json</code> file:</p>
            <pre><code>{`{
  "servers": {
    "my-tools": {
      "type": "stdio",
      "command": "node",
      "args": ["./mcp-server/dist/index.js"]
    }
  }
}`}</code></pre>
            <p>Once configured, you can reference MCP tools in Copilot Chat using the <code>#</code> symbol to select available tools. Copilot will call the tools as needed during your conversation.</p>
        </AccordionContent>
    </AccordionItem>
    <AccordionItem value="item-4">
        <AccordionTrigger className="text-lg text-left">Building a Custom MCP Client</AccordionTrigger>
        <AccordionContent className="prose dark:prose-invert max-w-none text-base">
            <p>You can build your own MCP client using the SDK. This is useful for embedding MCP in custom applications, scripts, or automation pipelines.</p>
            <pre><code>{`import { Client } from "@modelcontextprotocol/sdk/client/index.js";
import { StdioClientTransport } from "@modelcontextprotocol/sdk/client/stdio.js";

// Create a client that connects to a server
const transport = new StdioClientTransport({
  command: "node",
  args: ["path/to/server.js"],
});

const client = new Client({ name: "my-app", version: "1.0.0" });
await client.connect(transport);

// List available tools
const tools = await client.listTools();
console.log("Available tools:", tools);

// Call a tool
const result = await client.callTool({
  name: "count_words",
  arguments: { text: "Hello world" },
});
console.log("Result:", result);`}</code></pre>
            <p>Custom clients give you full programmatic control. You can build CLI tools, web APIs, cron jobs, or any application that needs to interact with MCP servers.</p>
        </AccordionContent>
    </AccordionItem>
    <AccordionItem value="item-5">
        <AccordionTrigger className="text-lg text-left">Antigravity Configuration</AccordionTrigger>
        <AccordionContent className="prose dark:prose-invert max-w-none text-base">
            <p>Antigravity (Google DeepMind's AI coding agent) supports MCP servers through its settings UI and workspace configuration:</p>
            <p><strong>Setting up:</strong></p>
            <ol>
                <li>Open Antigravity Settings → MCP Servers</li>
                <li>Add your server with command, args, and environment variables</li>
                <li>The server's tools become available in your conversations immediately</li>
            </ol>
            <p><strong>Key advantage:</strong> Antigravity runs MCP servers alongside its built-in tools (file editing, terminal, browser, image generation). This means your custom MCP server can work in concert with Antigravity's native capabilities.</p>
            <p><strong>Example workflow:</strong> You build a Google Sheets MCP server. Now you can tell Antigravity: "Read the sales data from the Q4 sheet, analyze it, and write a summary to the README." Antigravity uses the Sheets MCP to read data, its own reasoning to analyze, and its file system tools to write the result.</p>
        </AccordionContent>
    </AccordionItem>
    <AccordionItem value="item-6">
        <AccordionTrigger className="text-lg text-left">Per-Project vs Global Configuration</AccordionTrigger>
        <AccordionContent className="prose dark:prose-invert max-w-none text-base">
            <p>MCP servers can be configured at two levels, and choosing correctly matters for team workflows:</p>
            <p><strong>Global configuration:</strong></p>
            <ul>
                <li>Stored in user-specific locations (<code>~/Library/Application Support/Claude/claude_desktop_config.json</code>)</li>
                <li>Available in every conversation/project</li>
                <li>Best for: personal utility servers (weather, calculator, personal notes)</li>
                <li>Drawback: not shared with team, requires manual setup per machine</li>
            </ul>
            <p><strong>Per-project configuration:</strong></p>
            <ul>
                <li>Stored in the project repo (<code>.cursor/mcp.json</code>, <code>.vscode/mcp.json</code>)</li>
                <li>Committed to Git. team members get the same tools automatically</li>
                <li>Best for: project-specific servers (project database, deployment tools, internal APIs)</li>
                <li>Security: use environment variable references, not raw secrets in config files</li>
            </ul>
            <p><strong>Pattern for teams:</strong> Create a <code>.cursor/mcp.json</code> with your project's servers. Add a <code>.env.example</code> documenting the required environment variables. Team members clone the repo, copy <code>.env.example</code> to <code>.env</code>, fill in their credentials, and they're connected.</p>
        </AccordionContent>
    </AccordionItem>
</Accordion>
