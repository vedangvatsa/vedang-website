        <AccordionContent className="prose dark:prose-invert max-w-none text-base">
            <p>LLMs can be fantastic debuggers. When you have an error, provide the full context, what you call the "debugging trifecta":</p>
            <ol>
                <li><strong>The code</strong> that caused the error.</li>
                <li><strong>The full error message</strong> and traceback.</li>
                <li><strong>Your intent:</strong> what were you trying to do?</li>
            </ol>
            <blockquote className="border-l-4 border-primary/50 pl-4 italic my-2">"The Python code below gives a `NameError: name 'toUpperCase' is not defined`. I'm trying to convert a string to uppercase. Please debug the code, fix it, and suggest any other improvements you see."</blockquote>
            <p>Often, the model will not only fix the immediate bug but also identify other issues and suggest more robust, efficient code.</p>
            <p><strong>Code Review Prompts:</strong> You can also use LLMs as a code reviewer by asking it to analyze your code for potential issues:</p>
            <blockquote className="border-l-4 border-primary/50 pl-4 italic my-2">"Review the following Python function for: 1) Security vulnerabilities (SQL injection, etc.), 2) Performance issues, 3) Edge cases that could cause errors. Suggest fixes for each issue found."</blockquote>
        </AccordionContent>

            <p><strong>Documentation Generation:</strong></p>
            <blockquote className="border-l-4 border-primary/50 pl-4 italic my-2">"Add comprehensive JSDoc documentation to every function in the following code. Include @param, @returns, @throws, and @example tags."</blockquote>
