<ToolboxExplainer />

The tool you choose determines how fast you can build. There is no single perfect tool. There is only the right tool for the specific project you are building right now. 

This module breaks down the modern AI coding toolkit. We categorize tools into two main types: Cloud IDEs and Local IDEs.

<SectionLabel label="Choosing Your Environment" />

Your first decision is where the code will live while you work on it.

<SectionLabel label="A Tour of Your Options" />

Here is a breakdown of the most effective tools available right now and when to use them.

<Accordion type="single" collapsible className="w-full mt-4 space-y-2">
    <AccordionItem value="lovable" className="border rounded-lg bg-card px-4">
        <AccordionTrigger className="text-lg font-semibold hover:no-underline">
            1. Lovable.dev & v0 by Vercel: The Visual Builders
        </AccordionTrigger>
        <AccordionContent className="text-base text-muted-foreground pt-1">
            **Best For:** Fast, beautiful, visual web apps and UI components.
            
            **How it works:** You describe what you want in natural language, and the AI generates a working UI in real-time. **Lovable** generates full-stack apps with Supabase integration, auth flows, and one-click deployment. **v0 by Vercel** generates production-ready React components with Tailwind CSS — you can copy the code directly into your Next.js project. Both are browser-based, so there's zero local setup. The trade-off: you have less control over architecture decisions compared to local tools.
        </AccordionContent>
    </AccordionItem>
    
    <AccordionItem value="cursor-windsurf" className="border rounded-lg bg-card px-4">
        <AccordionTrigger className="text-lg font-semibold hover:no-underline">
            2. Cursor & Windsurf: The AI-Native Editors
        </AccordionTrigger>
        <AccordionContent className="text-base text-muted-foreground pt-1">
            **Best For:** Developers who want AI integrated directly into their coding environment with full codebase awareness.
            
            **How it works:** Both are forks of VS Code with AI deeply integrated. **Cursor** has inline editing (Cmd+K), a chat panel that indexes your entire codebase for context-aware suggestions, and "Composer" mode for coordinated multi-file changes. **Windsurf** offers similar capabilities with its "Cascade" flow for autonomous edits. Both support MCP servers, meaning you can connect them to databases, APIs, and custom tools. The key advantage over cloud IDEs: they run locally, so you get full filesystem access, custom extensions, and no vendor lock-in on your code.
        </AccordionContent>
    </AccordionItem>
    
    <AccordionItem value="replit" className="border rounded-lg bg-card px-4">
        <AccordionTrigger className="text-lg font-semibold hover:no-underline">
            3. Replit: The All-in-One Workshop
        </AccordionTrigger>
        <AccordionContent className="text-base text-muted-foreground pt-1">
            **Best For:** Full-stack apps, backend services, bots, and rapid prototyping with instant deployment.
            
            **How it works:** Replit gives you a full cloud development environment — code editor, terminal, package manager, and live preview in one tab. Its AI agent can generate entire projects from a single prompt and deploy them to a public URL with one click. Replit handles infrastructure (hosting, SSL, domains) so you never touch a server. The trade-off: your code lives on Replit's servers, and the free tier has compute limits.
        </AccordionContent>
    </AccordionItem>
    
    <AccordionItem value="copilot" className="border rounded-lg bg-card px-4">
        <AccordionTrigger className="text-lg font-semibold hover:no-underline">
            4. GitHub Copilot & Codex: The Inline Assistants
        </AccordionTrigger>
        <AccordionContent className="text-base text-muted-foreground pt-1">
            **Best For:** Developers who want AI autocomplete inside their existing VS Code or JetBrains setup without switching editors.
            
            **How it works:** **Copilot** predicts and autocompletes code as you type, trained on billions of lines of open-source code. It's the lowest-friction AI tool — install the extension and it starts suggesting. **Codex** (OpenAI's latest) goes further: it can execute multi-step coding tasks asynchronously in a sandboxed cloud environment, like a remote pair programmer. The key distinction from Cursor: Copilot is an add-on to your existing editor, while Cursor replaces the editor entirely.
        </AccordionContent>
    </AccordionItem>
    
    <AccordionItem value="agents" className="border rounded-lg bg-card px-4">
        <AccordionTrigger className="text-lg font-semibold hover:no-underline">
            5. Claude Code & Antigravity: The Autonomous Agents
        </AccordionTrigger>
        <AccordionContent className="text-base text-muted-foreground pt-1">
            **Best For:** Large-scale refactors, complex multi-file changes, and autonomous project-level work.
            
            **How it works:** **Claude Code** (Anthropic) runs in your terminal as an interactive agent. You describe a task, and it reads your codebase, writes code, runs shell commands, and iterates on errors autonomously. **Antigravity** (Google DeepMind) is an agentic coding assistant that operates inside your IDE — it can read your project, edit files, run commands, browse the web, and execute multi-step tasks with deep codebase understanding. Both represent the frontier of AI coding: instead of autocompleting one line, they autonomously execute entire features. The trade-off vs. Cursor: higher autonomy means less granular control over each edit.
        </AccordionContent>
    </AccordionItem>

    <AccordionItem value="ollama" className="border rounded-lg bg-card px-4">
        <AccordionTrigger className="text-lg font-semibold hover:no-underline">
            6. Ollama: The Local Intelligence
        </AccordionTrigger>
        <AccordionContent className="text-base text-muted-foreground pt-1">
            **Best For:** Privacy-sensitive projects, offline development, and running AI without API costs.
            
            **How it works:** Ollama lets you download and run open-source models (Llama 3, Mistral, Qwen, DeepSeek) directly on your machine. Your code never leaves your hardware — no API calls, no cloud dependency, no per-token billing. You need decent hardware (16GB+ RAM, Apple Silicon or a good GPU) to run larger models smoothly. Many local IDEs like Cursor and Continue can connect to Ollama as a backend, so you get the same AI-assisted workflow but fully offline and private.
        </AccordionContent>
    </AccordionItem>
</Accordion>

    <AccordionItem value="q2">
        <AccordionTrigger className="text-lg font-medium text-left">What makes Claude Code and Antigravity different from Cursor?</AccordionTrigger>
        <AccordionContent className="text-base text-muted-foreground border-l-4 border-primary/50 pl-4">
            **Autonomy level.** Cursor is an AI-assisted editor — you're still driving, and the AI suggests or edits when asked. Claude Code and Antigravity are autonomous agents: you describe a goal ("add authentication to this app"), and they independently plan the changes, edit multiple files, run tests, fix errors, and iterate until the task is done. Think of Cursor as power steering and Claude Code/Antigravity as self-driving.
        </AccordionContent>
    </AccordionItem>
    <AccordionItem value="q3">
        <AccordionTrigger className="text-lg font-medium text-left">Why would someone choose Ollama over a cloud model?</AccordionTrigger>
        <AccordionContent className="text-base text-muted-foreground border-l-4 border-primary/50 pl-4">
            **Privacy and cost.** Cloud models like GPT-4o and Claude send your code to external servers — a non-starter for companies handling sensitive IP, health data, or financial code. Ollama runs everything locally: zero data leaves your machine, zero API costs (after the one-time model download), and zero internet dependency. The trade-off is that local models are typically less capable than frontier cloud models, and you need strong hardware (Apple Silicon Mac or NVIDIA GPU) to run them at usable speeds.
        </AccordionContent>
    </AccordionItem>
</Accordion>
