<ToolboxExplainer />

The tool you choose determines how fast you can build. There is no single perfect tool. There is only the right tool for the specific project you are building right now. 

This module breaks down the modern AI coding toolkit. We categorize tools into two main types: Cloud IDEs and Local IDEs.

<SectionLabel label="Choosing Your Environment" />

Your first decision is where the code will live while you work on it. Think of this like choosing a workshop.

<SectionLabel label="A Tour of Your Options" />

Here is a breakdown of the most effective tools available right now and when to use them.

<Accordion type="single" collapsible className="w-full mt-4 space-y-4">
    <AccordionItem value="lovable" className="border rounded-lg bg-card px-4">
        <AccordionTrigger className="text-lg font-semibold hover:no-underline">
            1. Lovable.dev & v0 by Vercel: The Visual Builders
        </AccordionTrigger>
        <AccordionContent className="text-base text-muted-foreground pt-2">
            **Best For:** Fast, beautiful, visual web apps and UI components.
            
            **How it works:** You describe what you want in chat, and the AI builds the user interface visually in real-time. Lovable connects easily to databases like Supabase, while v0 specializes in generating clean React components with Tailwind CSS that you can copy-paste into other projects.
        </AccordionContent>
    </AccordionItem>
    
    <AccordionItem value="cursor-windsurf" className="border rounded-lg bg-card px-4">
        <AccordionTrigger className="text-lg font-semibold hover:no-underline">
            2. Cursor & Windsurf: The AI-Native Editors
        </AccordionTrigger>
        <AccordionContent className="text-base text-muted-foreground pt-2">
            **Best For:** Developers who want AI integrated directly into their coding environment.
            
            **How it works:** Both are forks of VS Code with AI built in at every level. Cursor has inline editing (Cmd+K), a chat panel that reads your codebase, and "Composer" mode for multi-file changes. Windsurf offers similar capabilities with its "Cascade" flow for autonomous edits. Both support MCP servers for extending AI capabilities.
        </AccordionContent>
    </AccordionItem>
    
    <AccordionItem value="replit" className="border rounded-lg bg-card px-4">
        <AccordionTrigger className="text-lg font-semibold hover:no-underline">
            3. Replit: The All-in-One Workshop
        </AccordionTrigger>
        <AccordionContent className="text-base text-muted-foreground pt-2">
            **Best For:** Apps with complex logic, backend services, bots, and learning how code works.
            
            **How it works:** Replit gives you a full computer running in the cloud. It includes a code editor, a command line terminal, and a live web preview. Its AI can generate entire projects from a single prompt and deploy them to the internet with one click.
        </AccordionContent>
    </AccordionItem>
    
    <AccordionItem value="copilot" className="border rounded-lg bg-card px-4">
        <AccordionTrigger className="text-lg font-semibold hover:no-underline">
            4. GitHub Copilot & Codex: The Inline Assistants
        </AccordionTrigger>
        <AccordionContent className="text-base text-muted-foreground pt-2">
            **Best For:** Developers who want AI autocomplete inside their existing VS Code or JetBrains setup.
            
            **How it works:** Copilot predicts and autocompletes code as you type. Codex (the latest iteration) can execute multi-step coding tasks asynchronously in a sandboxed environment. Both integrate into editors you already use without requiring a switch.
        </AccordionContent>
    </AccordionItem>
    
    <AccordionItem value="agents" className="border rounded-lg bg-card px-4">
        <AccordionTrigger className="text-lg font-semibold hover:no-underline">
            5. Claude Code & Antigravity: The Autonomous Agents
        </AccordionTrigger>
        <AccordionContent className="text-base text-muted-foreground pt-2">
            **Best For:** Making changes across many different files at once or building complex local architectures.
            
            **How it works:** These are autonomous agents running in your terminal (Claude Code) or as background systems (Antigravity). They read your entire project folder and can perform large multi-file refactors, execute tests, and commit changes to version control with minimal supervision.
        </AccordionContent>
    </AccordionItem>

    <AccordionItem value="ollama" className="border rounded-lg bg-card px-4">
        <AccordionTrigger className="text-lg font-semibold hover:no-underline">
            6. Ollama: The Local Intelligence
        </AccordionTrigger>
        <AccordionContent className="text-base text-muted-foreground pt-2">
            **Best For:** Privacy-focused development, offline coding, and fine-tuning custom models.
            
            **How it works:** Ollama allows you to run powerful open-source models (like Llama 3, Mistral, or Qwen) locally on your own hardware. Your code never leaves your machine, making it essential for enterprise security and offline vibe coding.
        </AccordionContent>
    </AccordionItem>
</Accordion>
