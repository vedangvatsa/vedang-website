If blockchain is the immutable ledger, smart contracts are what make it programmable and intelligent. This module explores how these pieces of code are transforming blockchain from a simple database into a global, permissionless computer.

<Accordion type="single" collapsible className="w-full mt-4">

        <AccordionTrigger className="text-lg">What are Smart Contracts?</AccordionTrigger>
        <AccordionContent className="prose dark:prose-invert max-w-none text-base">
            A smart contract is simply a program that runs on the blockchain. It's a self-executing contract where the terms of the agreement between parties are written directly into code. They run exactly as programmed without any possibility of downtime, censorship, fraud, or third-party interference.
            
            > **Analogy: A Vending Machine.** A vending machine is a physical smart contract. You put in money (input), select an item (condition), and the machine automatically dispenses the item (output). There's no cashier needed; the logic is baked into the machine. Smart contracts are digital vending machines for any kind of agreement or application.

            Here's a simplified example of what a smart contract looks like conceptually:

            ```
            IF user sends 0.1 ETH to this contract
            AND the item is still available
            THEN transfer the digital item to the user's wallet
            AND mark the item as sold
            ELSE refund the 0.1 ETH to the user
            ```

            The key insight is that once this logic is deployed to the blockchain, **nobody can change it**, not even the person who created it. The rules are enforced by code, not by trust.
        </AccordionContent>
    </AccordionItem>
    <AccordionItem value="item-2">
        <AccordionTrigger className="text-lg">Key Properties and Capabilities</AccordionTrigger>
        <AccordionContent className="prose dark:prose-invert max-w-none text-base">
            * **Autonomous:** Once deployed to the blockchain, they run automatically when their conditions are met, without needing anyone to trigger them. A lending protocol will automatically liquidate a loan if the collateral value drops below a threshold, 24/7, with no human intervention.
            * **Immutable:** The code of a smart contract cannot be changed after it's deployed. This ensures that the rules of the game are fixed and cannot be altered by a single party. (Note: some contracts use "proxy patterns" to allow upgrades, but the original code itself remains on-chain forever.)
            * **Transparent:** The code is visible to everyone on the blockchain, allowing anyone to verify its logic and functionality. This radical transparency means that anyone can audit a DeFi protocol's code before trusting it with their money.
            * **Composable ("Money Legos"):** This is one of the most powerful features. Because smart contracts are open and live on the same network, developers can build new applications by combining existing ones, like snapping together Lego blocks. A new DeFi protocol can integrate a pre-existing stablecoin and a decentralized exchange to create a completely new product without asking for permission.

            This composability accelerates innovation rapidly and is the driving force behind the growth of the Web3 ecosystem.
        </AccordionContent>

    <AccordionItem value="item-4">
        <AccordionTrigger className="text-lg">Smart Contract Languages and Security</AccordionTrigger>
        <AccordionContent className="prose dark:prose-invert max-w-none text-base">
            Smart contracts are written in specialized programming languages designed for blockchain environments:

            * **Solidity:** The most popular language, used for Ethereum and EVM-compatible chains (Polygon, Arbitrum, Base). Its syntax is similar to JavaScript, making it accessible to web developers.
            * **Rust:** Used by Solana and other high-performance blockchains. Rust's strong memory safety guarantees help prevent certain classes of bugs.
            * **Move:** Developed by Facebook's Diem project, now used by Aptos and Sui. It's designed specifically for safe asset handling.

            **Security is critical.** Because smart contracts are immutable and often control large amounts of money, bugs can be catastrophic. The 2016 "DAO Hack" exploited a vulnerability in a smart contract to drain $60 million worth of ETH. Today, the industry has matured significantly with:

            * **Professional Auditing Firms:** Companies like Trail of Bits, OpenZeppelin, and Certora specialize in reviewing smart contract code before deployment.
            * **Bug Bounty Programs:** Protocols offer rewards (often $100K+) for hackers who find and responsibly disclose vulnerabilities.
            * **Formal Verification:** Mathematical proofs that a contract's code matches its intended behavior, the highest level of security assurance.
